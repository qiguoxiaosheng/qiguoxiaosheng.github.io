Java中的引用类型

Java中有4中引用类型，分别为:

  • 强引用(Strong Reference)，默认的引用类型，强应用之所以为强，是因为如果一个对象到GC Roots强引用可达，只要某个对象有强引用与之关联，这个对象永远不会被回收，即使内存不足，JVM抛出OOM，也不会去回收。 输出结果为：
  • 软引用（Soft Reference）软引用就是把对象用SoftReference修饰一下，当我们需要从软引用对象获得修饰的对象，只要get一下就可以了：如果GC后，内存充足，那么这个对象会停留在内存中，当内存不足时垃圾回收器才会回收这些软引用的修饰的对象。软引用比较适合用作缓存，当内存足够，可以正常的拿到缓存，当内存不够，就会先回收缓存占用的空间。 输出结果为：
  • 弱引用（Weak Reference）弱引用的使用和软引用类似，只是关键字变成了WeakReference，弱引用的特点是不管内存是否足够，只要发生GC，都会被回收，使用WeakReference，垃圾回收器会帮你来决定引用的对象何时回收并且将对象从内存移除（当没有任何强引用到弱引用对象时）。弱引用在比如ThreadLocal、WeakHashMap等很多地方都有用到。 输出结果为：
  • 虚引用（Phantom Reference）十分脆弱，它的唯一作用就是当其指向的对象被回收之后，自己被加入到引用队列，用作记录该引用指向的对象已被销毁
